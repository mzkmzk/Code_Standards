# Code_Standards

## 1. 约定规则

1. 项目名,类名均采用下划线首字母大写
2. 非类的文件名采用下划线字母小写
3. 文件夹名均下划线小写
4. 顺序命名 先实体分类->再功能分类, 例如在活动Controller中,编写修改活动分享信息 share_setting_update
5. 命名采用动宾结构

## 2. Ajax数据传输规范

1. 每个接口必须包含`err_code`和`err_msg`
2. 每个页面初始必须直入直出,ajax只为提升用户交互体验

## 3. 教训

1. 不要投机取巧,eg: 参数名为id,其实具体意思是子订单id,在url中没体现出是子订单Id,所以参数要命名为具体的子订单ID

## 4. GitBook 编写规范

1. 目录跟+号对应,大标题也用#
